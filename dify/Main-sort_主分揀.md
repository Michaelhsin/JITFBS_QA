【Main-sort 主分揀】

---

## 流程
刷區域 QR code → 刷 Device ID → 刷商品 SKUID → PDA 顯示工位、播種牆亮燈 → 拍燈 → 關箱（步驟1. 刷區域 QR code → 步驟2. 刷 TO 單）→ 綁箱成功。

---

## 常見錯誤與處理方式

[錯誤1]
PDA錯誤訊息／異常狀況：此區域已超過三人
可能原因：區域人員超過三人
處理方式：等待空位後再選擇區域

[錯誤2]
PDA錯誤訊息／異常狀況：無區域資料
可能原因：系統內無此區域設定
處理方式：在 Seatalk live issue 回報 @FBS PM 協助

[錯誤3]
PDA錯誤訊息／異常狀況：此揀貨箱尚未有訂單，此為空箱
可能原因：該箱已播種完成
處理方式：播下一箱

[錯誤4]
PDA錯誤訊息／異常狀況：該揀貨箱區域錯誤，請移轉至其他區域
可能原因：區域錯誤
處理方式：開啟分貨模組，再次刷分貨確認正確位置

[錯誤5]
PDA錯誤訊息／異常狀況：撿貨箱號中的門市全部漏設
可能原因：門市／工位／mainsort 模組資料缺漏
處理方式：Seatalk live issue 回報 @FBS PM 協助

[錯誤6]
PDA錯誤訊息／異常狀況：因區域解除佔用或人員未選區域
可能原因：未先刷區域 QR Code 就先刷 Device ID
處理方式：先刷區域 QR Code 再進行作業

[錯誤7]
PDA錯誤訊息／異常狀況：X 無訊息
可能原因：少貨或商品放錯箱
處理方式：解箱並交給退貨組，依 SOP-01 處理

[錯誤8]
PDA錯誤訊息／異常狀況：箱內尚有產品，PDA 亦有顯示數量但刷不過
可能原因：此為多貨、取消、或箱內商品無可匹配訂單
處理方式：解箱並交給退貨組處理

[錯誤9]
PDA錯誤訊息／異常狀況：工位不存在
可能原因：未設定播種工位
處理方式：Seatalk live issue 回報 @FBS PM

[錯誤10]
PDA錯誤訊息／異常狀況：商品已完成播種，請刷讀其他商品
可能原因：商品重刷
處理方式：丟入物流箱

[錯誤11]
PDA錯誤訊息／異常狀況：PDA 顯示工位後，播種牆未亮燈
可能原因：播種牆異常
處理方式：請依 SOP-02 操作

[錯誤12]
PDA錯誤訊息／異常狀況：前一次播種尚未拍燈
可能原因：播種牆卡燈或未拍燈
處理方式：依 SOP-03 處理

[錯誤13]
PDA錯誤訊息／異常狀況：TO 已重複綁定
可能原因：TO 單重複刷讀
處理方式：更換新 TO 單

[錯誤14]
PDA錯誤訊息／異常狀況：TO 單門市錯誤或訂單被取消
可能原因：TO 單門市與工位不符
處理方式：依 SOP-04 處理

## SOP 詳細說明

SOP-01：PDA 尚顯示待播種商品數量，但箱內無商品
說明：
1. 直接解箱。
2. 箱內商品放入 mainsort 異常箱，交給退貨組。

SOP-02：PDA 顯示工位後，播種牆未亮燈
說明：
1. 現場重啟控制器。  
2. 若無法處理，在 Seatalk live issue 回報 @vendor 協助。  
3. 若頻繁發生：紀錄工位 → Seatalk live issue 回報 @vendor。

SOP-03：播種牆卡燈／未拍燈處理
說明：
1. 檢查現場是否有未拍燈。  
2. 若燈未滅 → 登入 Web 系統 → 控制設定 → 手動解除滅燈。  
3. 若仍無法 → 在 Seatalk live issue 回報 @vendor 協助。  
4. 若頻繁發生 → 記錄工位 → 回報 @vendor。

SOP-04：TO 單門市錯誤處理
說明：
1. 確認 TO 單與工位的門市是否一致。  
2. 若一致 → 商品放入 Main-sort 異常箱 → 交給 LRT。  
3. 若不一致 → 回報 @FBS PM。
