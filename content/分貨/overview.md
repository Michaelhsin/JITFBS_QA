# 分貨

```flow
開啟分貨模組 → 刷 Device ID → 系統分配區域工位 → 播種牆亮燈 → 確認商品正確 → 投入對應工位 → 拍燈 → 分貨完成
```

---

## 常見錯誤與處理方式

| 錯誤訊息 | 可能原因 | 處理方式 |
|-----------|-----------|-----------|
| 此揀貨箱尚未有訂單，此為空箱 | ① 拿空箱刷分貨。<br>② 箱內有商品但系統顯示空箱。 | ① 若確實為空箱，可直接使用。<br>② 若箱內有商品但顯示空箱 → 直接「解箱」，箱內商品放 **Main-sort 異常箱**，交給 **LRT**。 |
| 刷讀內容非揀貨箱號格式，請重新刷讀正確的揀貨箱號 | 刷到錯誤的條碼（非箱號） | 重新刷讀正確的揀貨箱號。 |
| 無區域工位模組資料，請協助至 WEB 後台確認目前啟用模組是否有建立 | Web 系統上未設定工位模組（顯示於播種牆設定） | 在 Seatalk **live issue** 回報 **@FBS PM** 協助設定工位。 |
| 楊梅：請推至南部 Presort 播種牆<br>安南：請推至北部 Presort 播種牆 | 系統偵測到跨倉 C2C RTS（錯倉轉運） | ① 若在楊梅倉 → 交給「轉安南倉」<br>② 若在安南倉 → 交給「轉楊梅倉」 |

## SOP 詳細說明

<details>
<summary><strong>SOP-01：空箱顯示異常</strong></summary>

1. 確認箱內是否真的有商品。  
2. 若箱內為空 → 可直接繼續使用。  
3. 若箱內有商品 →  
　(1) 解箱 →  
　(2) 商品放 **Main-sort 異常箱** →  
　(3) 交 **LRT** 處理。  
</details>

<details>
<summary><strong>SOP-02：無工位模組</strong></summary>

1. 進入 Web 後台 → 「播種牆管理」。  
2. 檢查當前倉別是否已建立工位模組。  
3. 若無 → Seatalk **live issue** 標記 **@FBS PM** 協助新增模組。  
</details>

<details>
<summary><strong>SOP-03：錯倉轉運 (C2C RTS)</strong></summary>

- 若 PDA 顯示：「楊梅: 請推至南部 Presort 播種牆」 → 將箱交給 **轉安南倉**。  
- 若 PDA 顯示：「安南: 請推至北部 Presort 播種牆」 → 將箱交給 **轉楊梅倉**。  
- 回報後標記狀態為「倉別轉運中」。  
</details>
