【分貨】

---

## 流程
開啟分貨模組 → 刷 Device ID → PDA 顯示對應 Mainsort 播種牆編號 → 推至分貨區地面對應編號 → 疊箱 → 待滿落時，推至對應 Mainsort 播種牆 → 分貨完成。

---

## 常見錯誤與處理方式

[錯誤1]
PDA錯誤訊息／異常狀況：此揀貨箱尚未有訂單，此為空箱
可能原因：
① 拿空箱刷分貨。
② 箱內有商品但系統顯示無資料。
處理方式：
① 若確實為空箱，可直接使用。
② 若箱內有商品但顯示空箱 → 直接解箱，箱內商品放入 Main-sort 異常箱 → 交給退貨組（見 SOP-01）。

[錯誤2]
PDA錯誤訊息／異常狀況：刷讀內容非揀貨箱號格式，請重新刷讀正確的揀貨箱號
可能原因：刷到錯誤的條碼（非箱號）
處理方式：重新刷讀正確的揀貨箱號。

[錯誤3]
PDA錯誤訊息／異常狀況：無區域工位模組資料，請協助至 WEB 後台確認目前啟用模組是否有建立
可能原因：播種牆系統上未設定工位（在 Web 上顯示）
處理方式：在 Seatalk live issue 回報 @FBS PM 協助設定工位（見 SOP-02）。

[錯誤4]
PDA錯誤訊息／異常狀況：楊梅：請推至南部 Presort 播種牆／安南：請推至北部 Presort 播種牆
可能原因：系統偵測到要跨倉的 C2C RTS（錯倉轉運）
處理方式：
① 若在楊梅倉 → 交給「轉安南倉」。
② 若在安南倉 → 交給「轉楊梅倉」（見 SOP-03）。

## SOP 詳細說明

SOP-01：空箱顯示異常
說明：
1. 確認箱內是否真的有商品。
2. 若箱內為空 → 可直接繼續使用。
3. 若箱內有商品 →
　(步驟1) 解箱。
　(步驟2) 商品放入 Main-sort 異常箱。
　(步驟3) 交退貨組處理。

SOP-02：無工位模組
說明：
1. 進入 Web 後台 → 「播種牆管理」。  
2. 檢查當前倉別是否已建立工位模組。  
3. 若無 → Seatalk live issue 標記 @FBS PM 協助新增模組。

SOP-03：錯倉轉運 (C2C RTS)
說明：
- 若 PDA 顯示：「楊梅: 請推至南部 Presort 播種牆」 → 將箱交給 Presort 組 C2C RTS to S（轉安南倉）。  
- 若 PDA 顯示：「安南: 請推至北部 Presort 播種牆」 → 將箱交給 Presort 組 C2C RTS to N（轉楊梅倉）。
